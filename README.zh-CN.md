# Clipboard Keeper

[English](README.md)

一个小巧的本地 Windows 剪贴板历史工具，支持文本和图片。

## 功能

- 在应用运行时捕获文本和图片剪贴板变化。
- 保留最近 100 条不重复记录。
- 在列表中显示图片缩略图，并在预览区显示较大图片。
- 双击图片预览时打开更大的图片查看窗口。
- 显示所选图片的本地存储路径，并可在文件资源管理器中打开。
- 根据描述长度自动调整主列表每一行的高度，最多显示三行。
- 支持按 `All`、`Text`、`Image` 或 `Star` 过滤主列表。
- 在主列表中使用图标区分项目类型。
- 当主过滤器设为 `Image` 时，会放大预览区域。
- 支持收藏重要项目；已收藏项目按最新收藏时间置顶。
- 可通过 `Copy Selected` 按钮将选中项目复制回剪贴板。
- 可为每条记录添加可编辑的 `Name`，方便快速识别。
- 支持多选并删除多条记录，也可以清空所有保存记录。
- 可通过 `Exit` 按钮明确退出应用。
- 最小化到 Windows 系统托盘，并继续在后台监听剪贴板。
- 点击托盘图标会显示紧凑快捷面板，双击托盘图标会恢复完整窗口。
- 紧凑快捷面板同步显示主列表的过滤器、收藏状态、类型图标、名称和描述。
- 历史记录仅保存在本机的 `%LOCALAPPDATA%\JiraAceClipboardManager` 下。
- 使用当前 Windows 用户账户加密保存的文本和图片文件。
- 构建时会生成并嵌入本地应用图标，用来表达可复用的文本和图片剪贴板内容。

## 安全说明

- 不访问网络。
- 不需要管理员权限。
- 不注入浏览器。
- 不使用键盘钩子，也不记录按键。
- 不注册开机自启动。
- 使用标准 Windows 剪贴板 API。
- 保存的历史记录使用 Windows DPAPI 加密，因此其他 Windows 用户不能直接读取存储目录中的内容。

由于 Windows 不会向普通应用暴露过去的系统剪贴板历史，本应用只能从它开始运行的时刻起建立历史记录。应用启动时会捕获一次当前剪贴板内容，之后记录后续的剪贴板变化。

## 构建

在仓库根目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\build.ps1
```

可执行文件会生成在：

```text
bin\ClipboardKeeper.exe
```

构建脚本还会生成并嵌入：

```text
assets\ClipboardKeeper.ico
```

## 运行

打开：

```text
bin\ClipboardKeeper.exe
```

需要记录剪贴板历史时，请保持窗口运行。复制敏感内容前，可以使用 `Pause` 暂停记录。

当你最小化窗口或点击 `X` 关闭按钮时，应用会隐藏到系统托盘，而不是直接退出。点击托盘图标可打开紧凑快捷面板，双击托盘图标可恢复完整窗口。可在紧凑面板中点击 `Main Window` 恢复完整窗口，或点击 `Exit` 完全退出。

要给项目命名，请在主列表中选中它并点击 `Edit Name`，或按 `F2`。
要置顶项目，请选中它并点击 `Star`；再次点击 `Unstar` 可取消置顶。
